De ecuaciones en una pantalla a un mapa que puedes tocar

Ayer formalizamos el diagrama causal en variables y relaciones. Hoy en la mañana esas relaciones se vuelven código que simula el tiempo paso a paso; en la tarde, esa simulación se convierte en el primer prototipo visual de tu videojuego.

CH-15 · DE FLECHAS A NÚMEROS

Un diagrama causal dice "qué afecta a qué". Una simulación necesita "cuánto"

Tu diagrama del Notebook 1 tiene flechas como +población → −área verde. Para que la computadora pueda avanzar el tiempo, cada flecha necesita convertirse en una ecuación: un número que sube o baja un poquito en cada paso, según el estado actual del sistema.

# Cada variable del diagrama causal se vuelve un atributo del estado
class EcosistemaUrbano:
    def __init__(self):
        self.poblacion = 10000
        self.area_verde = 0.65      # fracción del terreno, 0 a 1
        self.contaminacion = 0.10

    def paso(self, tasa_crecimiento, presion_urbana):
        # cada flecha del diagrama = una línea de código
        self.poblacion += self.poblacion * tasa_crecimiento
        self.area_verde -= presion_urbana * (self.poblacion / 100000)
        self.contaminacion += (1 - self.area_verde) * 0.02
        self.area_verde = max(0, self.area_verde)
Esto es exactamente lo que construirás en el Notebook 2 con NumPy: un objeto con estado que cambia en cada "tick" del tiempo, siguiendo las mismas relaciones que ya dibujaste como flechas.

CH-18 · DE PROTOTIPO A MECÁNICA DE JUEGO

Un prototipo responde al mouse; un juego responde a decisiones con consecuencias

El mapa que acabas de pintar ya es interactivo, pero todavía no es un juego: no hay turnos, ni presupuesto, ni consecuencias que se acumulen con el tiempo. Esa es exactamente la diferencia entre el Notebook 3 (prototipo de interfaz, hoy) y el Notebook 4 (escenarios y mecánicas, mañana).

Guarda esta pregunta para mañana: ¿qué decisión sencilla (una herramienta, un botón) podría tener la consecuencia más interesante en tu ciudad? Ese suele ser el germen de la mecánica principal de un buen videojuego de simulación.
